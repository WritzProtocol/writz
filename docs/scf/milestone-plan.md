## Tranche #0 — 10% (~$9,200)

**Theme:** Prove the core primitive works.

**Deadline:** August 2026

### Deliverable 1: SPV contract verifies a real Bitcoin mainnet transaction on Soroban testnet

**Verification:** SCF reviewer can run the following Stellar CLI command and observe a successful result:

```bash
stellar contract invoke \
  --id CDYQRO6PZ55A3AMJQBHDEUUCQTSVHHRWQW7WSDX7CBX6FQ2NLKYCPLVC \
  --network testnet \
  -- verify_transaction \
  --headers '[...]' \
  --merkle_proof '[...]' \
  --tx_index 42 \
  --raw_tx '...' \
  --min_confirmations 6
```

The headers and Merkle proof are fetched from a real confirmed Bitcoin mainnet transaction and submitted to the Soroban testnet contract. The contract returns `{ txid, block_hash, confirmations: 6 }`.

**What has already been achieved (before grant disbursement):**

## Tranche #1 — 20% (~$18,400)

**Theme:** Full deposit → borrow flow on testnet.

**Deadline:** October 2026

### Deliverable 1: PrivateLend v1 on Soroban testnet

End-to-end flow working on testnet:
1. User deposits BTC to a P2WSH address generated by the frontend
2. After 6 confirmations, submits SPV proof to PrivateLend (which calls the SPV contract)
3. USDC becomes available to borrow against the verified BTC collateral
4. User borrows USDC to their Stellar wallet
5. User repays USDC debt
6. Protocol emits `repay_full` event; backend co-signs P2WSH release

**Verification:** SCF reviewer can observe the end-to-end sequence on Stellar testnet using provided transaction IDs.

### Deliverable 2: ZK privacy stub working on testnet

Phase 2 ZK privacy is not complete by Tranche #1, but the **interface** is live:
- Deposit creates a Poseidon commitment hash stored on-chain
- Borrow/repay updates the commitment (no ZK proof yet — full ZK in Tranche #2)
- Positions identified by commitment hash, not by Stellar address

This establishes the privacy-ready data model before full circuit integration.

### Deliverable 3: End-to-end testnet demo video

10–15 minute demo of the full deposit → borrow → repay cycle using real Signet BTC and Soroban testnet USDC.

### Deliverable 4: STRIDE threat model (draft)

First version of the STRIDE security analysis published in docs (`scf/stride-threat-model.md`). See the [STRIDE document](stride-threat-model.md) for the current draft.

## Tranche #2 — 30% (~$27,600)

**Theme:** Privacy complete. Frontend live. Audit-ready.

**Deadline:** December 2026

### Deliverable 1: All three Circom circuits implemented and tested

- **Deposit circuit** (~280 constraints): Poseidon commitment creation
- **Borrow/repay circuit** (~10,500 constraints): ZK proof of valid state transition preserving collateral ratio
- **Liquidation circuit** (~9,000 constraints): ZK proof of undercollateralization without revealing amounts

Each circuit:
- Compiled to R1CS
- Test witnesses generated and verified
- Proving time measured in browser (target: <8 seconds for borrow/repay)

### Deliverable 2: Trusted setup ceremony executed

Powers of Tau Phase 2 for each of the three circuits:
- Ceremony transcript published publicly
- Multiple independent verifiers confirm transcript integrity
- Final `.zkey` files and verification keys published

### Deliverable 3: Frontend v1 live on testnet (app.writz.io)

Functional testnet frontend with:
- Xverse wallet connection (BTC side)
- Freighter wallet connection (Stellar side) via Stellar Wallets Kit
- Deposit flow: generate P2WSH address → detect confirmation → submit SPV proof
- Borrow/repay flow with ZK proof generation in-browser
- Position dashboard (shows health ratio, current debt)

### Deliverable 4: Audit Bank application submitted

Application to the Soroban Security Audit Bank submitted to SDF with:
- Code deployed on testnet with extensive tests (141 tests across all contracts)
- STRIDE threat model complete
- Self-service tooling scans complete (Mythril/Slither)
- Dataflow diagram produced

**Theme:** Mainnet. Real users. Real money.

**Deadline:** March 2027

### Deliverable 1: Mainnet deployment

Both Soroban contracts (SPV + PrivateLend) deployed on Stellar mainnet with:
- Security audit completed with zero critical or high findings
- All audit findings remediated and re-verified
- Mainnet contract addresses published in docs

### Deliverable 2: First real BTC deposit processed

A real Bitcoin mainnet transaction processed through the full stack:
1. Real BTC sent to a Writz P2WSH address on Bitcoin mainnet
2. SPV proof submitted to Soroban mainnet contract
3. USDC borrowed against the verified BTC collateral

**Verification:** Bitcoin transaction ID + Stellar transaction ID published as public proof.

### Deliverable 3: Public launch with $50K TVL cap

Launch with a conservative TVL cap (removes existential risk from a Day 1 exploit). Cap is lifted after:
- 30 days of clean mainnet operation
- Audit findings confirmed remediated in production

### Deliverable 4: DeFiLlama listing

Writz Protocol listed on DeFiLlama as a Stellar DeFi protocol with verified on-chain TVL. DeFiLlama listing is independently verifiable and provides a credible, third-party TVL metric.

### Deliverable 5: SPV SDK open-sourced as standalone crate

The Bitcoin SPV verification logic packaged as a standalone Rust crate (`writz-spv`) publishable on crates.io. Documentation and examples showing how any Soroban protocol can integrate Bitcoin transaction verification.
